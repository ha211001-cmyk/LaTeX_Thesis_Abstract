\section{\LaTeX の概要}
Microsoft Wordみたいなものです。テキストファイルでいろいろ書きます。習うより慣れよ。
\begin{itemize}
    \item 図や式の番号を自動管理してくれる (参照)
    \item 参考文献リストが扱いやすい
    \item 長い文章でも重くならない (Wordよりは)
    \item ソフトが落ちない (ただの平文テキストファイルなので)
    \item きれい
    \item 図が画面外に消えていなくならない
\end{itemize}
などのメリットがありますが覚えることが多いのが難点。\textbf{ある程度網羅的なサンプルがあればコピペで作れるのでこのテンプレを全員で育てましょう。}

\subsection{OverleafとCloudLatex}
どちらもオンラインLatex環境ですが、Overleafは無料プランではコンパイル時間に制限があります。使い勝手はOverleafのほうが良い印象ですが、Cloud Latexはコンパイル時間に制限がない、VS Codeと連携できたりとメリットがあります。報告書であればOverleafが勝手が良いとは思いますが、どちらか一つしか使わないのであればCloud Latex推奨です。Overleafで使う場合にはlatexmkrcファイルと、左上メニューからコンパイラ: Latexの設定が必要です。どちらにしても新規プロジェクトからzipファイルのアップロードができます。

\subsection{ローカル\LaTeX 環境の構築}
卒論などの長文では、Gitでバージョン管理したほうがよく、またlatexdiff-vcで差分ファイルを作成できると便利です。Cloud Latexはローカルファイルと同期させることもできるので実現可能ですが、latexdiff-vcは使えないのでローカル環境を整えたほうが良い場合も多いです。最も手軽なVS Code-Docker環境のセットアップを紹介します。Windows環境でインストールすべきは
\begin{itemize}
  \item VS Code
  \item Docker Desktop
\end{itemize}
です。また、VS CodeのExtensionsとして、
\begin{itemize}
  \item Remote Development (4つのextensionsのセット、必須なのはDev Containers)
  \item LaTeX Workshop
  \item Remote Explorer (必須ではない)
\end{itemize}
をインストールしましょう。これで準備完了です。使い方は
\begin{enumerate}
  \item VS Codeを起動し、フォルダをD\&Dしてワークスペースを開く
  \item 左下の``$><$''から、Reopen in containerを押してDockerコンテナのシェルに入る
  \item main.texを開いて\TeX のところからBuild LaTeX project - Recipe: latexmkかまたは、Terminalからlatexmk main.texと打つ。どのファイルを選択していてもよいので、コマンド打つ方が早い。
\end{enumerate}
で、PDF出力までされると思います。SyncTeXに対応しており (LaTeX Workshopの設定が必要かも)、PDFをctrl-クリックで該当のソースへ移動、ソースからはTexタブのView LaTeX PDF - Refresh...で該当のPDFの場所に移動ができます。ちなみに、.devcontainerフォルダにはdockerの設定ファイル、.vscodeフォルダにはLaTeX Workshopの設定ファイル (ワークスペース限定) が入っています。gitignoreはgitの無視するファイルを設定。latexmkrcはlatexmkコマンドの設定。
% latexmk -synctex=1 -silent -pdfdvi main.tex

Gitを使うなら、
\begin{itemize}
  \item Git
  \item GitHub Desktop
\end{itemize}
くらい入れれば良い。Clone, Commit, Fetch (Pull), Push, Checkoutくらいを使えれば便利さを感じることができると思います。

\subsection{章と節、その他}

章は\textbackslash section\{章タイトル\}、節は\textbackslash subsection\{節タイトル\}と書く。subsubsectionもある。
基本的にテキストで書いた順番に並べられるが、図表は自動で配置を決定する。空白行を入れると段落が変わる。数式や図を入れるときに空白行を無駄に入れてしまうのに注意。
逆に空白行さえ入れなければ段落は変わらず、改行もなく続けて表示される。バックスラッシュ2個で改行できるが、基本的に使わないほうがよい。

\section{表と図}

表は表\ref{tab:my_label}のように、
\begin{table}[b]
    \centering
    \caption{Captios should be in English.}
    \begin{tblr}{
      width = { 1.0\linewidth },
      colspec = {X[1] X[1]}, % 比率指定
      % colspec = {>{\centering}p{.1\linewidth} p{.8\linewidth}}, % 幅指定
      % row{1} = { valign = h },
      }
      \hline
      1 & 2 \\
      abc & xyz \\
      \hline
    \end{tblr}\label{tab:my_label}
\end{table}
図は図\ref{fig:my_figure}のように書く。
\begin{figure}[b]
    \centering
    \includegraphics[width=0.4\linewidth]{fig/wind_power.JPG}
    \caption{Captios should be in English.}
    \label{fig:my_figure}
\end{figure}
並べたい場合はminipageを使う (図\ref{fig:my_figure2}, \ref{fig:my_figure3})。バックスラッシュ2つで改行を入れて縦に並べることもできる。オプションHでtexファイルで書いた位置に画像を入れる。基本的にはt、b、tb (上か下) 推奨。配置にこだわってもうまくいかないことが多いので気にしないということも重要。
\begin{figure}[b]
  \begin{minipage}[b]{0.47\linewidth}
    \centering
    \includegraphics[width=0.8\linewidth]{fig/AngularVelocity.pdf}
    \caption{Caption}\label{fig:my_figure2}
  \end{minipage}
  \hspace{0.02\linewidth} % ここで隙間作成
  \begin{minipage}[b]{0.47\linewidth}
    \centering
    \includegraphics[width=0.8\linewidth]{fig/AngularVelocity.pdf}
    \caption{Caption}\label{fig:my_figure3}
  \end{minipage}
\end{figure}
\begin{figure}[b]
  \begin{minipage}[b]{0.47\linewidth}
    \centering
    \includegraphics[width=0.8\linewidth]{fig/AngularVelocity.pdf}
    \subcaption{Caption}\label{fig:my_figure4-a}
  \end{minipage}
  \hspace{0.02\linewidth} % ここで隙間作成
  \begin{minipage}[b]{0.47\linewidth}
    \centering
    \includegraphics[width=0.8\linewidth]{fig/AngularVelocity.pdf}
    \subcaption{Caption}\label{fig:my_figure4-b}
  \end{minipage}
  \caption{Captios should be in English.}\label{fig:my_figure4}
\end{figure}
キャプションをまとめたいときはsubcaption (図\ref{fig:my_figure4}, \ref{fig:my_figure4-a})。
2段組み中の1段組みは図\ref{fig:my_figure5}のようにfigure*環境を使う。
\begin{figure*}[tb]

    \includegraphics[width=0.2\linewidth]{fig/AngularVelocity.pdf}

    \label{fig:my_figure5}
\end{figure*}

\subsection{数式環境}

equation環境:
\begin{equation}
    y = f(x)
\end{equation}
align環境:
\begin{align}
    y_1 &= f_1(x_1,~x_2) \\
    y_2 &= f_2(x_1,~x_2)
\end{align}
aligned環境:
\begin{equation}
    \begin{aligned}
        y_1 &= f_1(x_1,~x_2) \\
        y_2 &= f_2(x_1,~x_2)
    \end{aligned}
\end{equation}
bmatrix環境:
\begin{equation}
    A = 
    \begin{bmatrix}
        a & b \\
        c & d
    \end{bmatrix}
\end{equation}
長い数式はfigure*環境の中にequation環境等を入れる。その際キャプションはつけない。

\section{報告書の章立て}
研究報告は以下の構成で書くこと。
\begin{enumerate}
    \item 概要

    報告の内容を簡潔にまとめる。

    \item 研究計画
    
    しっかり研究計画を練ること。

    \item 導入

    以下の項目について書く。

    \begin{itemize}
        \item 動機付け: 研究の背景や意義は？
        \item 関連研究: 既存研究の紹介と解決すべき課題は？
        \item 目的: なにに取り組むのか？
    \end{itemize}

    \item 本題

    適宜、章を分ける。

    \item 結論
    
\end{enumerate}
